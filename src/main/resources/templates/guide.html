üöÄ Getting Started
Welcome to BISEN! This guide will help you get started with testing your APIs.
1
Access the Application
Open BISEN in your browser. The default URL is http://localhost:2000. You'll see the main request builder interface.
2
Familiarize Yourself with the Interface
The interface consists of:
  • Left Sidebar: Collections and saved requests
  • Top Navigation: Access to different sections (Home, History, Collections, Environments, etc.)
  • Request Builder: Main area to configure and send API requests
  • Response Section: View API responses below the request builder
3
Make Your First Request
Try a simple GET request:
  1. Select GET from the method dropdown
  2. Enter a URL (e.g., https://jsonplaceholder.typicode.com/posts/1)
  3. Click the Send button
  4. View the response below
üì° Making API Requests
Learn how to configure and send different types of API requests.
1
Select HTTP Method
Choose from: GET, POST, PUT, DELETE, PATCH, HEAD, or OPTIONS.
2
Enter URL
Type the complete API endpoint URL. You can use variables like {{baseUrl}}/api/users (see Environments section for details).
3
Configure Request Parameters
Use the tabs to configure different aspects:
  • Params: URL query parameters (e.g., ?page=1&limit=10)
  • Authorization: Set up authentication (Basic, Bearer Token, API Key, etc.)
  • Headers: Add custom HTTP headers
  • Body: Request body for POST, PUT, PATCH requests
  • Settings: Request name, timeout, environment selection
4
Send Request
Click the Send button (or press Enter). The response will appear below with:
  • Status code (200 OK, 404 Not Found, etc.)
  • Response time in milliseconds
  • Response size
  • Response body (formatted JSON if applicable)
  • Response headers
üí° Pro Tip
Use the Format button in the Body tab to automatically format JSON. The Validate button checks if your JSON is valid before sending.
üìÅ Collections
Organize your API requests into collections for better management.
1
Create a Collection
Go to the Collections page and click Create Collection. Give it a name and optional description (e.g., "User Management APIs").
2
Add Requests to Collection
When saving a request, select the collection from the dropdown in the Settings tab. You can also add existing saved requests to collections using the + button on saved requests.
3
View Collection Requests
Collections appear in the left sidebar. Click to expand and see all requests in that collection. Click any request to load it into the request builder.
üí° Tip
Collections help organize APIs by feature, service, or environment. For example, create separate collections for "Authentication", "User Management", and "Payment APIs".
üíæ Saved Requests
Save frequently used requests for quick access.
1
Save a Request
After configuring a request, click the Save button. Optionally enter a name in the Settings tab. The request will appear in the left sidebar.
2
Load a Saved Request
Click any saved request in the left sidebar to load it into the request builder. You can modify and send it, or save changes.
3
Duplicate Requests
Click the ‚ßâ button on any saved request to create a copy. Useful for creating variations of the same request.
4
Search Requests
Use the search box in the left sidebar to quickly find requests by name, method, or URL.
üåç Environments & Variables
Manage different environments (Development, Staging, Production) and use variables to make requests dynamic.
üéØ Why Use Environments?
Environments let you switch between different API endpoints, API keys, and configurations without manually changing URLs and headers. Perfect for testing across Development, Staging, and Production.

Creating Environments

1
Access Environment Manager
Click Environments in the top navigation menu, or click Manage Environments in the Settings tab on the Home page.
2
Create New Environment
Click the Create Environment button. Enter a name like:
  • Production
  • Staging
  • Development
  • Local Dev
3
Add Variables
In the environment modal, click + Add Variable for each variable you want to define. Common variables include:
  • baseUrl - Base API URL
  • apiKey - API authentication key
  • token - Authentication token
  • timeout - Request timeout value
// Example: Production Environment Variables { "baseUrl": "https://api.example.com", "apiKey": "prod_key_12345", "version": "v1" } // Example: Development Environment Variables { "baseUrl": "http://localhost:3000", "apiKey": "dev_key_67890", "version": "v1" }

Managing Variables

1
Edit Environment Variables
On the Environments page, click the ‚úèÔ∏è Edit button on any environment card. You can add, modify, or remove variables. Click Save when done.
2
Duplicate Environments
Use the üìã Duplicate button to create a copy of an environment. Great for creating similar environments with slight variations.
3
Set Default Environment
Environments can be marked as default. The default environment's variables are automatically available when no environment is selected.

Using Variables in Requests

1
Select an Environment
In the Settings tab, select an environment from the dropdown. You can also use the environment selector in the top navigation bar.
2
Use Variable Syntax
Use {{variableName}} syntax anywhere in your request:
  • URL: {{baseUrl}}/api/users
  • Headers: X-API-Key: {{apiKey}}
  • Body: {"url": "{{baseUrl}}/webhook"}
// Example Request URL with Variables {{baseUrl}}/api/{{version}}/users?page=1 // Example Header with Variable Authorization: Bearer {{token}} // Example Body with Variables { "endpoint": "{{baseUrl}}/api/notify", "apiKey": "{{apiKey}}" }
3
Variable Substitution
When you send a request, BISEN automatically replaces all {{variableName}} placeholders with the actual values from the selected environment. Variables are substituted in:
  • Request URL
  • Query parameters
  • Request headers
  • Request body
üí° Best Practice
Create separate environments for each deployment stage. Use consistent variable names across environments (e.g., always use baseUrl) so you can easily switch between environments without changing your requests.
üìù Example Workflow
  1. Create "Production" environment with baseUrl: "https://api.prod.com"
  2. Create "Staging" environment with baseUrl: "https://api.staging.com"
  3. Create a request with URL: {{baseUrl}}/users
  4. Switch environments to test the same request against different servers
üîê Authorization
Secure your API requests with various authentication methods.
üîë Basic Auth
Username and password authentication. Credentials are Base64 encoded and sent in the Authorization header.
üé´ Bearer Token
Token-based authentication. Enter your token and it will be sent as Authorization: Bearer <token>.
üîë API Key
API key authentication. Specify the key value and header name (default: X-API-Key).
üîí Digest Auth
Digest authentication with username and password. Handles the authentication challenge automatically.
üí° Pro Tip
Store authentication tokens and API keys as environment variables for better security and easier management. Use {{apiKey}} in headers instead of hardcoding values.
üì• Importing Swagger/OpenAPI
Import API specifications to automatically create collections and requests.
1
Access Import Page
Go to Import Swagger from the top navigation menu.
2
Import from URL or File
You can import in two ways:
  • From URL: Enter the Swagger/OpenAPI JSON URL
  • From File: Upload a Swagger/OpenAPI JSON or YAML file
3
Automatic Collection Creation
BISEN automatically:
  • Creates collections based on API tags
  • Generates saved requests for each endpoint
  • Sets up request methods, URLs, and parameters
  • Configures request bodies based on schemas
üìú Request History
View and manage all your past API requests.
1
Access History
Click History in the top navigation to view all executed requests.
2
View Request Details
Each history entry shows:
  • HTTP method and URL
  • Status code and response time
  • Request and response data
  • Timestamp
3
Reuse History Requests
Click on any history entry to view full details. You can copy the request configuration to create a new request or save it for future use.
üí° Tips & Best Practices
Get the most out of BISEN with these helpful tips.
üìù Naming Conventions
Use descriptive names for requests and collections. Example: "GET User Profile" instead of "Request 1".
üîÑ Environment Strategy
Create environments for each deployment stage. Use consistent variable names across environments for easy switching.
üìÅ Organization
Group related APIs into collections. Create collections by feature, service, or team for better organization.
üîç Search & Filter
Use the search box in the sidebar to quickly find requests. Filter by method, URL, or collection name.
‚è±Ô∏è Timeout Settings
Set appropriate timeouts in the Settings tab. Default is 30 seconds. Adjust based on your API response times.
üìã Copy Responses
Use the Copy button in the response section to quickly copy response data for use in other tools or documentation.
